# PROJECT_BRIEF.md

## Basic Information

- **Project Name**: dotfiles-manager
- **Project Type**: cli
- **Primary Goal**: A CLI tool to sync, backup, and manage dotfiles across multiple machines with support for profiles and encrypted secrets
- **Target Users**: developers, system administrators, power users
- **Timeline**: 2 weeks
- **Team Size**: 1

## Functional Requirements

### Key Features (MVP)

- Initialize and sync dotfiles to a git repository
- Create symlinks from dotfiles repo to home directory
- Support multiple profiles (work, home, server)
- Encrypt sensitive files (API keys, tokens)
- Cross-platform support (macOS, Linux)
- Backup and restore functionality

### Nice-to-Have Features (v2)

- GUI for profile management
- Cloud sync without git
- Dotfile templates with variables
- Integration with 1Password/Bitwarden

## Technical Constraints

### Must Use

- Python 3.11+
- Click
- GitPython
- cryptography (for encryption)

### Cannot Use

- Node.js
- Rust

## Other Constraints

- Must work without root/sudo for user dotfiles
- Should handle 1000+ dotfiles efficiently
- Must not overwrite existing files without confirmation
- Encryption must use industry-standard algorithms (AES-256)
